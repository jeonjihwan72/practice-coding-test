# 백준 코딩 테스트

카테고리 : 브루트 포스

원문 링크 : [백준 코딩 테스트 - 브루트 포스](https://www.acmicpc.net/step/22)

## 브루트 포스란?

브루트 포스(Brute Force) 알고리즘은 문제 해결을 위해 가능한 모든 경우의 수를 시도하는 가장 단순하지만 강력한 알고리즘이다.
완전탐색(Exhaustive Search)이라고도 불리며, 가능한 모든 경우의 수를 전부 탐색하며 정답을 찾는다.

### 장점

1. 간단하고 직관적이다 : 구현 방법이 간단하고 표현식이 직관적이기에 이해하기 쉽다.

2. 보편적인 적용이 가능하다 : 여러 종류의 문제에 적용되어도 문제를 해결할 수 있다. 특히나 입력의 크기가 작거나 가능한 해의 수가 제한적인 경우에 유용하게 사용할 수 있다.

3. 항상 정확한 결과를 도출한다 : 가능한 모든 경우의 수를 시도하기 때문에 항상 정확한 결과를 도출할 수 있다. 최적의 해가 주어지며 동시에 검증된 답안을 얻을 수 있다.

### 단점

1. 대량으로 입력을 받을 시 효율이 떨어진다 : 실행할 때 모든 경우의 수를 시도하기 때문에 입력 크기에 따라 실행 시간이 기하급수적으로 증가한다. 그리고 경우에 따라 불필요한 계산이 발생할 가능성이 있다. 이에 따라 어떠한 경우에는 불필요한 중복 계산이나 검증된 부분을 다시 계산해야 할 수 있다.

2. 메모리 비효율성 : 브루트 포스 알고리즘은 가능한 모든 경우를 탐색하므로 메모리 사용량이 급격히 증가할 수 있다. 특히, 중간 결과를 저장하거나 중복 연산을 수행하면 불필요한 메모리 낭비가 발생한다. 문제의 크기가 커질수록 메모리 부족으로 인해 실행이 불가능할 수도 있다.

+ 1, 2번 공통적으로 불필요한 중복 연산과 재검증 과정을 문제로 들고 있다.
