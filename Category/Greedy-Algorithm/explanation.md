# 그리디 알고리즘 (Greedy Algorithm)

## 정의

- 그리디 알고리즘은 문제를 해결할 때, 각 단계에서 가장 최적이라고 판단되는 선택. 즉, 현재 순간에서 가장 좋은 선택을 하는 방식으로 작동하는 알고리즘이다.
- 이러한 접근 방식은 전체적인 최적 해를 보장하지는 않지만, 특정 문제에서는 효과적으로 최적 해를 찾을 수 있다.

## 장점

1. 단순한 구현 : 직관적이고 간단한 방식으로 문제를 해결할 수 있어 구현이 용이하다.

2. 빠른 수행 속도 : 일반적으로 동적 계획법(DP)보다 연산량이 적어 시간 복잡도가 낮은 경우가 많다.

3. 적은 메모리 사용 : 이전 단계의 결과만 고려하므로 추가적인 저장 공간이 적게 필요하다.

## 단점

1. 최적해를 보장하지 못함 : 항상 전역 최적해(Global Optimum)를 찾는 것은 아니며, 지역 최적해(Local Optimum)에 갇힐 위험이 있다.

2. 탐욕적인 선택이 실패할 수 있음 : 특정 문제에서는 과거의 선택이 이후의 해를 제한하여 최적해를 놓칠 수 있다.

3. 적용 가능한 문제의 제한 : 그리디 알고리즘이 반드시 정답을 보장하는 문제는 일부에 불과하며, 그리디 방법이 최적해를 보장하는지를 분석해야 한다.
