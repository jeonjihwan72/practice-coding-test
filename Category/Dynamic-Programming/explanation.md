# 백준 코딩 테스트

카테고리 : 동적 계획법 01 (Dynamic Programming, DP)

원문 링크 : [백준 코딩 테스트 - 동적 계획법 01](https://www.acmicpc.net/step/16)

## 정의

- 동적 계획법은 작은 부분 문제의 해결 결과를 저장하여 동일한 계산을 반복하지 않고 최적의 해를 구하는 알고리즘 기법이다. 

- 주로 최적 부분 구조(optimal substructure)와 중복 부분 문제(overlapping subproblems) 속성을 가지는 문제를 해결하는 데 사용된다.

## 장점

1. 시간 복잡도가 감소한다 : 부분 문제의 결과를 저장하여 불필요한 중복 연산을 줄여 시간 복잡도를 줄일 수 있다.

2. 최적의 해가 보장된다 : 그리디 알고리즘(Grid Algorithm)과 달리, 동적 계획법은 항상 최적의 해를 찾을 수 있다.

3. 다양한 문제를 해결할 수 있다 : 최단 경로, 배낭 문제, 문자열 편집 거리 등 다양한 문제에 적용할 수 있다.

## 단점

1. 공간 복잡도가 증가한다 : 부분 문제의 결과를 저장해야 하므로 메모리 사용량이 증가할 수 있다.

2. 설계가 복잡하다 : 문제를 적절한 부분 문제로 나누고, 점화식을 도출하는 과정이 어렵다.

3. 사용 조건 제한 : 최적 부분 구조와 중복 부분 문제 속성을 만족하지 않는 문제에는 사용할 수 없다.

## 종류
- 동적 계획법(DP)는 두 가지의 방식으로 구현할 수 있다.

1. 메모이제이션 (Top-Down Approach)
    - 주어진 문제를 재귀적으로 해결하면서, 계산된 값을 저장하여 중복 계산을 피하는 방식이다.
    - 함수를 호출할 때마다 값을 캐시하여 재사용한다.

2. 타뷸레이션 (Bottom-Up Approach)
    - 문제를 작은 부분 문제부터 해결해 나가면서, 점차적으로 전체 문제를 해결하는 방식이다.
    - 이 방식은 재귀없이 반복문을 사용하여 모든 값들을 채워 나가며, 최종 결과를 계산한다.
